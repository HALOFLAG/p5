## 三方視角總覽

| | 開發者 | 使用者 | 投資者 |
|---|---|---|---|
| **核心評價** | 中等難度，模組清晰 | 第 8 天才驚艷 | 自用 OK，商業 ROI 中等偏低 |

## 一、開發者視角（成本與難易度）

### 工時拆解

| 階段 | 內容 | 預估工日 |
|---|---|---|
| P1 | Electron 骨架、透明全螢幕、滑鼠穿透 | 1-2 |
| P2 | 對話氣泡 + 打字機 + 序列點擊 | 1-2 |
| P3 | uiohook + 觸發引擎 + 全螢幕偵測 + Logger | **3-5** |
| P4 | 台詞庫 schema + 人格切換 + Debug 面板 + **手寫 fallback 200 句** | **4-7** |
| P5 | Live2D 整合 + 拖曳 + 眼神 + 動作綁定 | 3-5 |
| P6 | 語音批次 + Audio + **嘴型同步** | 3-5 |
| P7 | Stage 1 pipeline + provider 抽象 + **prompt 工程** | **4-7** |
| **總計** | | **19-33 天** |

### 三個技術難點（會吃時間）

1. **Stage 1 的 prompt 工程**（P7）
   - 餵給 AI 的格式怎麼設計，才能讓它真的產出「個人化」而不是「敷衍化」的台詞？
   - 需要反覆迭代調 prompt + 寫校驗器（人格一致性、變數插值合法、長度範圍）。
   - 沒有現成範本可抄。

2. **動態冷卻參數調校**（P3）
   - 演算法本身好寫，但要調出「使用者覺得自然」的參數需長期實際使用測試。
   - 光寫程式碼不夠，要陪桌寵跑兩週才能調對。

3. **Live2D 嘴型同步**（P6）
   - FFT 或 RMS 算音量包絡 → 平滑 → 餵 `ParamMouthOpenY`。
   - 技術不難但「自然感」需多次調整係數。

### 不可控風險（第三方依賴）

- `uiohook-napi` 被防毒誤判 → 使用者首次啟動可能要白名單
- Electron 透明視窗在某些 Win11 + GPU 驅動閃爍 → 已知議題、無解
- 免費 Live2D 模型多帶授權限制（不可商用、需註明）

### 可重用性（正面）

模組解耦清楚：Logger / Trigger Engine / Stage 1 Generator 三個都是獨立模組，未來做別的桌寵或行為驅動工具時可整模組搬。

## 二、使用者視角（樂趣與複雜度）

### 體驗時序（這是這個架構的「決定性表現」）

| 階段 | 體驗 | 情緒 |
|---|---|---|
| 第 1 天 | 裝好開機，角色出現，講 fallback 句 | 普通桌寵 |
| 第 1-7 天 | fallback 逐漸重複（200 句撐 7 天，每天約 28 句…實際感受是「同一句又說了」）。設定面板顯示「資料收集中，剩 N 天」 | **耐心測試期** ⚠️ |
| 第 7 天 | 等 AI 跑 5-10 分鐘 batch | 期待 |
| **第 8 天起** | 角色突然會說：「**又週一早上 10 點開 VSCode 啦**」「**你這禮拜熬夜兩次了**」「**你連續打字 40 分鐘該休息了喔**」 | **哇靠它真的懂我** ⭐ |
| 第 8-35 天 | 1500 句 + 最近 50 句 ring buffer = 重複感極低 | 持續陪伴 |
| 第 35 天 | 提示「該重生新台詞了」→ 又一輪期待 | 像追番 |

### 核心樂趣 vs 競品

| 競品做不到的 | 為什麼 |
|---|---|
| runtime LLM 桌寵：只看當下對話 | **沒有「你這週熬夜兩次」這種跨時間觀察** |
| 預錄桌寵（VPet）：mod 內容是別人寫的 | **沒有為你個人觀察的內容** |
| 你這個專案 | 同時擁有「跨時間觀察」+「為你寫」 |

### 操作複雜度

| 場景 | 複雜度 |
|---|---|
| 安裝、開機、互動 | **低**（跟一般桌寵一樣） |
| 切人格、不打擾、音量 | 低（托盤 + 設定視窗） |
| 改 persona prompt、自寫台詞 | 中（要懂 JSON） |
| **每月按重生** | 低（一個按鈕） |

### 三個痛點

1. **第 1-7 天的耐心測試期** ← 最大風險
   - 200 句 fallback 對於每小時 5-10 次互動者，第 3-4 天就開始感覺重複
   - 緩解：fallback 寫到 **300-500 句**並涵蓋多種時段/情境

2. **AI 偶爾產出不符人格的句子**
   - 校驗器只能擋顯著問題，細微違和感擋不住
   - 緩解：archive 機制保留歷史版本，使用者可手動回滾或挑句

3. **隱私顧慮**
   - 「上傳詳細事件時序給 AI」對部分使用者是門檻（已接受此選項）
   - 緩解：提供「本地 LLM」provider 選項作為完全離線替代

## 三、投資者視角（成本與盈利點）

### 起步投入

| 項目 | 金額（¥） |
|---|---|
| 1 開發者 × 1.5 個月 | 40-60k |
| 商用 Live2D 模型授權（1-3 套） | 5-20k |
| AI API 開發/測試成本 | 1-3k |
| 視覺/UX 美化、設計資源 | 5-10k |
| 雜項（簽章、分發） | 2-5k |
| **合計起步** | **53-98k** |

執行期維運成本：**接近 0**（無後端、無資料庫、無雲端依賴）

### 差異化（USP）

| 比較對象 | 你的優勢 | 你的劣勢 |
|---|---|---|
| runtime LLM 桌寵（AI-tamago, Open-LLM-VTuber, petto） | 零延遲、零成本、零網路；個人化更深 | 對話多變度低 |
| 預錄桌寵（VPet 6k★, 桌寵製作大師） | 內容自動隨使用習慣更新 | 已有大量 mod 社群、IP 厚 |
| **「行為觀察 + 預生成」這個架構在桌寵領域沒有先例** | ← 這就是 USP | |

### 盈利點推演（按可行性排）

| # | 模式 | 可行性 | 評估 |
|---|---|---|---|
| 1 | **Free-to-play + AI 重生 DLC**（基礎免費，AI 個人化付費） | ★★★★ | 與架構天然契合：免費版用 fallback，付費版享個人化 |
| 2 | **B2B/IP 客製化**：賣框架給 VTuber 經紀公司、品牌方做粉絲桌寵 | ★★★★ | 單案 ¥10k-100k；2-3 案就回本 |
| 3 | **公司內部吉祥物方案**：賣給遊戲公司/IT 公司做員工陪伴 | ★★★ | 單案 ¥50k-500k，難開發業務但 ARPU 高 |
| 4 | **Live2D 模型市集**：與繪師分潤 | ★★ | 平台冷啟動難 |
| 5 | **聲優語音包**：付費高品質配音版本 | ★★ | 利基但獵奇 |
| 6 | **一次性 Steam 售賣 ¥50** | ★★ | 中文桌寵市場習慣低價/免費，1500-5000 套天花板 |
| 7 | **訂閱制 ¥10/月** | ★ | 桌寵不適合訂閱模式 |

### 商業風險

| 風險 | 影響 | 緩解 |
|---|---|---|
| 中文桌寵市場習慣低價/免費 | 一次性售賣天花板低 | 走 F2P + DLC 或 B2B |
| **想法易被複刻**（看你需求一個月就懂） | 護城河淺 | 護城河在「prompt 工程的累積調校」，這是時間 + 經驗，無法快速複刻 |
| 全域監聽 + 上傳事件時序 → GDPR/個資法 | EU 市場進入受阻 | 提供「全本地 LLM」選項解決 |
| 防毒誤判 hook | 退貨率 | 簽章 + 白名單說明 |
| Live2D 授權鏈 | 法律風險 | 商業化前清楚每個資產授權 |

### ROI 推估

| 情境 | 假設 | 結果 |
|---|---|---|
| 悲觀 | 1500 套 × ¥30 | ¥45k → **虧本** ¥10-50k |
| 基準 | 5000 套 × ¥50 | ¥250k → **回本 + 小賺** ¥150-200k |
| B2B 翻身 | 3 案 × ¥80k | ¥240k → 回本 + 中賺 ¥150k |
| F2P 模式 | 10k DAU × 5% 付費率 × ¥50/年 | ¥25k/年長尾收入 |

**直白結論**：作為純商業專案 ROI 中等偏低；**作為 IP 框架 / B2B 工具的 ROI 較高**。

## 總結與建議

### 真話總結

- **你原本說「自用軟體」，這個架構作為「自用」是極好的學習專案** — 涵蓋了 Electron、Live2D、全域 hook、行為日誌、AI batch generation、prompt 工程、隱私設計，是一條完整的端到端 pipeline，學完很值得。
- **但若想商業化**：純桌寵市場太擠且價格戰，你的 USP 在 B2B 客製化跟 F2P+DLC 模式上才有發揮空間。直接 Steam 上架賣 ¥50 收回開發成本可能性中等、爆款可能性低。

### 核心建議

> 先按「自用」做完，跑兩個月真實使用累積回饋再判斷要不要轉商業。
>
> - 如果 Stage 1 的 prompt 調得很好、生成的台詞有「真的懂我」的驚喜時刻 → **值得投入做商業化**
> - 如果調得普通沒驚喜 → **維持自用，不虧**
>
> 開發成本與你獲得的學習價值已大致對等，下行風險有限。

### 三個必須提早決定的事項（影響商業化路線）

1. **Live2D 模型路線**：自用免費模型 / 商用授權 / 委託繪製 — 影響 P5 預算
2. **AI provider**：完全離線（本地 LLM）/ 雲端（Claude / OpenAI）— 影響隱私敘事與成本
3. **是否考慮商業化**：影響從 P1 起的程式碼授權、資產來源、後端架構選型
