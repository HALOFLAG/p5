# 人物包設置指南（Persona Pack v3）

> p5 桌寵的每個 persona 都是一個 **self-contained 資料夾**，包含人格設定、對話庫、語音、立繪、Live2D 預留位置。
> 可以打包成 zip 分享，匯入後即用。

---

## 1. 什麼是 persona pack

一個 persona pack = `personas/<id>/` 整個資料夾。內含跑這個角色所需的**全部**資料：

```
personas/<id>/
├── persona.json                ★ 必要 — 人格設定 + voice + appearance + overrides
├── dialogues.json              ★ 必要 — 對話庫（觸發時抽句用）
├── dialogues-initial.json      ☆ 建議 — 種子對話（dialogues.json 不存在時 fallback）
├── voice-refs/                 ☆ 視需要 — TTS 參考音檔（要生語音才需要）
│   └── *-ref-ja.wav
├── appearance/
│   ├── static/
│   │   ├── main.png            ☆ 建議 — 預設立繪
│   │   └── expressions/        ☆ 視需要 — 9 個表情差分
│   │       ├── happy.png
│   │       ├── shy.png
│   │       ├── pout.png
│   │       └── ...
│   └── live2d/                 ☆ M8+ 預留 — Live2D Cubism 模型
│       └── README.md
├── voices/                     ⓘ 自動生 — 跑 TTS 後產生
│   └── <lang>/
│       ├── manifest.json
│       └── *.wav
└── voices-time/                ⓘ 自動生 — 時間音庫
    └── <lang>/
        ├── manifest.json
        └── hour_HH.wav / morning.wav / ...
```

★ = 必要 / ☆ = 可選 / ⓘ = 自動生成

## 2. persona.json 完整 schema

```jsonc
{
  "$schema": "v3",                          // ★ 必填，鎖定 schema 版本
  "id": "haiyin",                           // ★ 必填，唯一識別（限英數+底線+減號）
  "display_name": "海音",                   // ★ 必填，UI 顯示用

  // ── 人格設定（給 LLM 寫對話用）──
  "identity": {
    "age": "19",
    "background": "日本東京學生"            // 短描述
  },
  "personality": {
    "surface_impression": "...",
    "core_traits": ["黏膩", "獨佔", "...]"],
    "background": "..."
  },
  "speech_style": {
    "primary_tone": "甜膩中帶不安",
    "common_particles": ["呐", "啦", "..."],
    "common_kaomoji": ["♡", "✨"]
  },
  "expressions": {                          // 表情清單 + 觸發語感
    "idle": "...",
    "happy": "...",
    "shy": "...",
    "pout": "...",
    "annoyed": "...",
    "worried": "...",
    "embarrassed": "...",
    "sleepy": "...",
    "yandere": "..."
  },
  "voice_samples": [...],                   // LLM prompt 風格樣本（fallback）
  "preferences": {...},
  "behavioral_settings": {...},
  "linked_states": [...],
  "trigger_voice_guide": {                  // 各 category 風格指引
    "drag": "...",
    "long_idle": "...",
    "deep_night": "..."
  },

  // ── 視覺外觀 ──
  "appearance": {
    // ── 文字描述（給 LLM 寫文案參考）──
    "hair": "紫黑色長髮 + 雙馬尾",
    "eyes": "紫羅蘭色貓眼",
    "outfit_theme": "紫色蕾絲連衣裙",
    "color_palette": ["紫黑", "紫羅蘭", "粉紅"],
    "_keywords": "紫黑髮 / 雙馬尾 / 紫眼",

    // ── 靜態立繪 ──
    "static": {
      "main": "appearance/static/main.png",           // ★ 預設立繪
      "expressions": {                                  // 各表情差分（缺 = 用 main）
        "idle":        "appearance/static/expressions/idle.png",
        "happy":       "appearance/static/expressions/happy.png",
        "shy":         "appearance/static/expressions/shy.png",
        "pout":        "appearance/static/expressions/pout.png",
        "annoyed":     "appearance/static/expressions/annoyed.png",
        "worried":     "appearance/static/expressions/worried.png",
        "embarrassed": "appearance/static/expressions/embarrassed.png",
        "sleepy":      "appearance/static/expressions/sleepy.png",
        "yandere":     "appearance/static/expressions/yandere.png"
      }
    },

    // ── Live2D 預留（M8+ 啟用）──
    "live2d": {
      "enabled": false,                              // true 後 renderer 改載 Live2D
      "model_file": "appearance/live2d/model3.json",
      "motion_groups": {                              // 對應 expression 的動作群組
        "idle":  [],
        "happy": [],
        "pout":  []
      }
    }
  },

  // ── 語音設定（TTS 用）──
  "voice": {
    "ref_audio":      "voice-refs/haiyin-ref-ja.wav", // ★ 5-10s 參考音檔，相對 persona dir
    "ref_text":       "この私から逃げるつもり？面白い。", // ref 音檔的逐字稿
    "ref_lang":       "ja",                           // ref 音檔的語言
    "voice_lang":     "ja",                           // TTS 輸出語言（雙語架構用 ja）
    "additional_refs": [                              // 平均融合多 ref 音色更穩（同性別）
      "voice-refs/haiyin-ref-2.wav"
    ],
    "sampling": {                                     // 可選覆寫全域 sampling
      "temperature": 0.8,
      "top_k": 15,
      "top_p": 1.0
    }
  },

  // ── 時間語音文字客製（per language）──
  "time_voice_overrides": {
    "ja": {                                           // 預設字典在 src/main/voice-pipeline/time-voice-content.js
      "hour_14": "午後二時",
      "night":   "おやすみ"
    }
  }
}
```

## 3. 從頭建一個新 persona pack

### Step 1：建資料夾結構
```powershell
mkdir personas\my_persona
mkdir personas\my_persona\voice-refs
mkdir personas\my_persona\appearance\static\expressions
mkdir personas\my_persona\appearance\live2d
```

### Step 2：複製 [personas/haiyin/persona.json](../personas/haiyin/persona.json) 當模板
改 `id` / `display_name` / 各文字欄位。空 voice / appearance / time_voice_overrides 可以保留結構，內容之後填。

### Step 3：建 dialogues.json 骨架
13 個 category，每個先空陣列：

```jsonc
{
  "$schema": "v2",
  "persona_id": "my_persona",
  "categories": {
    "drag":            { "sequences": [] },
    "click_too_much":  { "sequences": [] },
    "long_idle":       { "sequences": [] },
    "continuous_use":  { "sequences": [] },
    "deep_night":      { "sequences": [] },
    "click_character": { "sequences": [] },
    "chatter":         { "sequences": [] },
    "boot_greet":      { "sequences": [] },
    "time_greet":      { "sequences": [] },
    "typing_burst":    { "sequences": [] },
    "click_burst":     { "sequences": [] },
    "app_switch":      { "sequences": [] },
    "game_enter":      { "sequences": [] }
  }
}
```

複製一份到 `dialogues-initial.json`。

### Step 4：放立繪
把預設立繪放 `appearance/static/main.png`（去背 PNG，建議 ≥ 800×1600）。
9 個表情差分（happy / shy / pout / annoyed / worried / embarrassed / sleepy / yandere / idle）有就放對應檔；沒放也 OK，會 fallback 到 main.png。

### Step 5：放 voice ref
要生語音才需要：
- 5-10 秒清晰人聲樣本 → `voice-refs/<id>-ref-<lang>.wav`
- 多 ref 融合（建議同性別）→ `voice-refs/<id>-ref-2.wav` 等
- 在 persona.json 的 `voice` 區塊填路徑 + ref_text 逐字稿

### Step 6：跑對話庫管理 UI 生內容
1. 啟動 app → 「對話庫管理」
2. Tab 📝 LLM Prompt → 選 persona + category → 跑 LLM
3. Tab 📋 批次匯入 → 貼草稿 → 套用
4. Tab 🔊 語音生成 → 設 voice ref（會寫進 persona.json）→ 批次生
5. Tab 🕐 時間語音 → 批次生時間音庫

## 4. 用既有 persona 當模板（最快）

```powershell
# 複製 haiyin → my_persona
cp -r personas\haiyin personas\my_persona

# 編輯 persona.json
#   - 改 id, display_name
#   - 改 identity / personality / speech_style 全部

#   - 改 voice.ref_text
#   - 清掉 dialogues.json + dialogues-initial.json 的 sequences
#   - 清掉 voices/ + voices-time/（讓 TTS 重新生）
#   - 替換 appearance/static/main.png
```

或用內建 wipe 腳本清空對話 + 語音：

```powershell
node scripts/wipe-all-content.js
```

## 5. 匯出 / 匯入 persona pack

### 匯出（分享給別人）
1. 對話庫管理 → header 「📂 開啟資料夾」
2. OS 檔案總管打開 `personas/<id>/`
3. **右鍵壓縮** 整個 `<id>/` 資料夾 → `<id>-pack.zip`
4. 傳給對方

⚠ 要排除：`_wipe-backup-*` / `_migrate-v3-*` / `_test_persona` 等 underscore 開頭資料夾（舊備份，可選擇性帶）

### 匯入（接收別人的 pack）
1. 把 `xxx-pack.zip` 解壓到任意位置
2. 對話庫管理 → header「📥 匯入 persona」
3. 選那個解壓後的資料夾（必須含 `persona.json`）
4. 自動：
   - 驗證 persona.json
   - 從 `persona.id` 取 id（或用資料夾名）
   - 檢查衝突（同 id 已存在 → 拒絕）
   - 複製到 `personas/<id>/`
5. **重啟 app** 才會載入新 persona（director 有 cache）

## 6. 必要 vs 可選 vs 自動生成

| 檔 / 資料夾 | 狀態 | 沒有時 |
|---|---|---|
| `persona.json` | ★ 必要 | 整個 persona 無法載入 |
| `dialogues.json` | ★ 必要 | 觸發時沒對話可抽 |
| `dialogues-initial.json` | ☆ 建議 | 沒有也可運作；建議跟 dialogues.json 同內容當種子 |
| `voice-refs/*.wav` | ☆ 視需要 | 沒設 voice.ref_audio 就不生語音；對話走純字幕模式 |
| `appearance/static/main.png` | ☆ 建議 | 沒有時 fallback 色塊 placeholder |
| `appearance/static/expressions/*` | ☆ 視需要 | 沒有時所有表情都用 main.png |
| `appearance/live2d/*` | ☆ M8+ | enabled=false 不影響 |
| `voices/<lang>/*` | ⓘ 自動 | 跑批次生成；缺 wav → 純字幕無聲 |
| `voices-time/<lang>/*` | ⓘ 自動 | 跑 Tab 6 批次生成 |
| `time_voice_overrides` | ☆ 可選 | 沒寫就用預設字典（src/main/voice-pipeline/time-voice-content.js）|

## 7. ID 命名規則

`persona.id` 必須符合：
- 只允許 `a-z` `0-9` `_` `-`
- 不可用中文 / 大寫字母 / 空格 / 特殊字元
- 慣例：英文小寫 + 短橫線（例 `haiyin` / `liss` / `lily-rose`）

理由：
- 用在 wav 檔名（`haiyin_chat_001_0.wav`）
- 用在 manifest key
- 用在資料夾名
- 跨平台 path safe

## 8. 9 個標準表情

| Expression | 中文 | 典型語境 |
|---|---|---|
| `idle` | 平靜 | 思考 / thought 內心戲多用 |
| `happy` | 開心 | 純粹開心、欣喜 |
| `pout` | 嘟嘴 | 輕嗔、小抱怨（無敵意）|
| `annoyed` | 不滿 | 明顯皺眉、不耐 |
| `embarrassed` | 害羞臉紅 | 真害羞（不是「無奈接受」）|
| `shy` | 害羞 | 害羞但內心開心 |
| `worried` | 擔心 | 體貼、關心 |
| `sleepy` | 睏倦 | 深夜場景 |
| `yandere` | 黑化 | 獨佔慾爆發（極少用，30 句最多 1-2 條）|

寫 LLM prompt 跟編輯 dialogues 時都用這 9 個英文 key。

## 10. 雙語對話格式

LLM 輸出格式（給 Tab 4 匯入用）：

```
[speech] 今天的你看起來特別專心呢。 | expression: happy | ja: 今日のあなた、すごく集中してるね。 | c: 2
[thought] （這人到底⋯⋯）| expression: pout | ja: （この人ってば⋯⋯）| c: 1
```

互動式 sequence（choice / binary）：

```
[speech] 想做什麼？ | expression: happy | ja: 何をしようかな？ | c: 3
@interactive {
  "interaction": "choice",
  "choices": [
    {
      "label": "想抱抱", "label_ja": "抱きしめて",
      "response": {
        "type": "speech", "expression": "shy",
        "text": "嗯⋯⋯不要在這裡⋯⋯♡",
        "voice_text": "ここではダメ⋯⋯",
        "next": "haiyin_click_hug_followup"   // 可選：鏈到下一個 sequence
      }
    }
  ]
}
```

`text` = 中文字幕（氣泡顯示）/ `voice_text` = 日文 TTS 來源。

## 11. 常見坑

| 問題 | 原因 | 解決 |
|---|---|---|
| 匯入後人格不出現 | 沒重啟 app（director cache）| 重啟 |
| 立繪不顯示 | `appearance/static/main.png` 路徑錯 | 檢查 persona.json + 檔案存在 |
| TTS 失敗「ref audio 找不到」 | `voice.ref_audio` 路徑相對錯誤 | 用相對 persona dir 的路徑（例 `voice-refs/x.wav`）|
| 語音音色怪 | `ref_lang` 跟 `voice_lang` 不同（跨語言）| 用同語言；ref 5-10s 清晰人聲 |
| Tab 6 時間語音預設 zh 不是 ja | UI lang 跟 persona.voice.voice_lang 自動同步 | 切人格時 Tab 6 / Tab 5 batch 的 lang 會自動切到該 persona 的 voice_lang；要看其他語言版本就手動切 |
| 對話沒出現 | dialogues.json 該 category 為空 | LLM 生 + Tab 4 匯入 |
| 觸發太頻繁/太少 | triggers.json 全域 cd 設定（不是 per-persona）| 改 config/triggers.json |
| 9 類分佈失衡 | LLM 沒按預設配比 | Tab 3 用預設或手動填配比 |

## 12. 跟引擎整合的關係

- **GPT-SoVITS api.py 啟動**：Tab 5「🚀 啟動引擎」一鍵啟動
- **API 設定**：`config/voice-config.json.engine_command` 設安裝路徑（全域，不在 persona 內）
- **persona 切換**：不需重啟引擎，跑 batch 自動帶該 persona 的 voice config

## 13. 進階：自製 Live2D persona

M8+ 預定支援，當前預留位置：

```
appearance/live2d/
├── model3.json         (Cubism 編輯器匯出)
├── *.moc3
├── physics3.json
├── textures/
├── motions/            (對應 motion_groups)
└── expressions/        (Cubism 表情，不是靜態 PNG)
```

在 `persona.json.appearance.live2d`：
- `enabled: true`
- `model_file` 指向 `model3.json`
- `motion_groups` 對應 trigger expression → motion 列表

實作時 renderer 改載 PIXI-Live2D-Display 跑模型。當前 enabled=false 仍用靜態 PNG。

## 15. 範例 pack（最小可運作版）

最少：

```
personas/my_minimal/
├── persona.json         (有 id / display_name / 基本 identity)
└── dialogues.json       (有 13 categories，sequences 全空陣列)
```

跑起來：
- 對話觸發時 director log「no sequences」warning（無致命錯）
- 不出對話也不出語音
- 桌寵 stage 顯示色塊 placeholder

逐步加：
1. 加 1 個 category 的 sequences → 該觸發開始有對話
2. 加 voice.ref_audio + ref_text → 對話有語音
3. 加 appearance/static/main.png → stage 顯示立繪
4. 加 expressions/*.png → 觸發 dialogue 時 stage 切表情
