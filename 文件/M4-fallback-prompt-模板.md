## §0 用法說明

### 怎麼用這份文件

1. **挑一份 prompt**（§2 / §3 共 10 份，每份對應一個人格的一個 category）
2. **整段複製** code block 內容到 LLM 對話框
3. **執行 → 收集輸出**（每輪約 30-50 句）
4. **重複 2-3 輪**讓不同 random seed 給多樣化結果（同 prompt 跑 3 次 ≈ 100-150 句草稿）
5. **篩選**（見 §4）→ 整理成 CSV → 之後用 `csv-to-dialogues.js` 自動轉 JSON

### 推薦的 LLM

| 選擇 | 適合情境 | 備註 |
|---|---|---|
| **本地 Gemma4** | 隱私為先 / 大量批次 | 見 [gemma4_api_guide.md](gemma4_api_guide.md)。中文輸出穩定度需驗證 |
| **Claude 3.5 Sonnet / Opus** | 中文細膩度高 / 病嬌語感準 | 推薦首選。海音的「中度病嬌」拿捏比 GPT 好 |
| **GPT-4 / GPT-4o** | 廣泛測試 | 中文偶爾會偏正式，要靠 voice_samples 拉回來 |
| **Gemini 1.5 Pro** | 免費額度大 | 中文 OK，但日系語氣需要強提示 |

### 預期產出量（每份 prompt 跑一輪）

- **每輪 30-50 句**（temperature 0.8-1.0 較有多樣性）
- 跑 3 輪 → 約 100-150 句草稿
- 篩選保留率約 40-60% → 每 category 落在 50-90 句
- 5 category × 平均 70 句 ≈ **350 句 / 人格**（超過 200-300 目標）

### 整理回 dialogues.json 的流程

```
LLM 輸出 (10 份 prompt × 3 輪)
  ↓
人工篩選 / 改寫 (§4 三標：保留 / 改寫 / 丟棄)
  ↓
整理成 CSV (見 §4 格式)
  ↓
[未來] csv-to-dialogues.js 自動補 sequenceId / 預設欄位
  ↓
合併進 personas/<id>/dialogues.json (v2 schema)
```

## §1 通用規則

### 變數白名單

僅這 5 個變數可在 `text` 內使用，運行時由 BubbleManager 替換：

| 變數 | 替換為 | 範例輸出 |
|---|---|---|
| `{time}` | 當前時刻（HH:MM） | 「都 02:14 了～」 |
| `{hour}` | 當前小時（0-23） | 「{hour} 點還醒著？」 |
| `{weekday}` | 中文星期 | 「週三也工作這麼認真～」 |
| `{usage_hours}` | 連續使用小時 | 「已經 3 小時了⋯⋯」 |
| `{window_title}` | 當前焦點視窗 | 「你在看 {window_title}？」（很少用，怕暴露隱私） |

**嵌入率建議：< 30%**（不是每句都要塞變數，太機械）。
其中 `{window_title}` 慎用（一份草稿最多 1-2 句）。

### 風格要求（兩人格通用）

**必須**：
- **1-2 句子**：太長違反桌寵氣泡 UI（每行最多 ~15-20 字）
- **不編號**：直接寫對白，不要「1. 你好」這樣的格式
- **不自指 AI**：禁用「身為 AI」「作為一個語言模型」等詞
- **角色一致**：海音用「我」/「你」；莉絲用「莉絲」/「主人」

**避免**：
- 過度華麗的修辭（「如同夜空中的繁星」這種）
- 解釋性語氣（「我這樣說是因為⋯⋯」）
- 長段獨白（超過 3 行）
- 表情貼圖過載（emoji 別超過 2 個 / 句）

### sequence type 的選用建議

dialogues.json v2 的 `type` 欄位有 5 種：

| type | 占比建議 | 用途 | 範例 |
|---|---|---|---|
| `speech` | **70%** | 一般對白（角色直接對使用者說話） | 「呐，怎麼點得這麼急～？」 |
| `thought` | **20%** | 內心獨白（用括號包） | 「（這個人在點什麼啊⋯⋯）」 |
| `narration` | 5% | 旁白 / 場景描述（少用） | 「（時鐘指針指向凌晨）」 |
| `whisper` | 5% | 低聲呢喃（特殊情境） | 「⋯⋯只看我一個人就好」 |
| `system` | 0% | M4 不用（只給系統訊息） | — |

**規則**：
- **internal monologue 用 `thought`**（括號包文字、不指向使用者）
- 海音的「嫉妒不安內心戲」適合 `thought`
- 莉絲的「擔心主人健康內心戲」適合 `thought`
- `whisper` 用於極黏膩 / 親密的場景（例：deep_night 凌晨 3 點）

### 預設欄位與覆寫

LLM 輸出時**只需要產生 `text` + 建議 `expression` + 可選 `type`**。
其他欄位（`persistence` / `interaction` / `auto_close_ms` / `sequenceId`）由整理腳本補。

預設值（如果 prompt 沒覆寫）：
```jsonc
{
  "type": "speech",
  "persistence": "transient",
  "interaction": "advance",
  "auto_close_ms": 12000
}
```

## §4 篩選與整理

### 篩選原則（三標）

LLM 一輪輸出 30-50 句，**保留率約 40-60%**。逐句標記：

| 標記 | 標準 | 動作 |
|---|---|---|
| ✅ **保留** | 風格對、句型自然、變數合理 | 直接收錄 |
| ✏️ **改寫** | 風格 80% 對但有小問題（多餘形容詞 / 變數位置怪 / 結尾不順） | 手動微調後收錄 |
| ❌ **丟棄** | 出戲（AI 自指）/ 重複 / 太極端 / 太正式 | 直接刪 |

**常見丟棄理由**：
- 海音：太極端（「殺了那個人」）/ 太正式（「請您注意休息」）
- 莉絲：忘記帶「最愛的主人♡」/「主人」/ 顏文字 / 誤稱「你」
- 通用：超過 2 句 / 含 AI 自指 / 跟現有句重複度高 / 變數使用錯誤

### CSV 整理建議格式

跑完 10 份 prompt × 3 輪後整理成這個格式（給未來 `csv-to-dialogues.js` 用）：

```csv
persona_id,category,type,text,expression,auto_close_ms,note
haiyin,click_too_much,speech,呐～怎麼點得這麼急嘛？,pout,,
haiyin,click_too_much,thought,（手指都在點別人嗎⋯⋯）,,5000,改寫自第3輪
haiyin,long_idle,speech,你⋯⋯剛才去哪了？{usage_hours} 分鐘呢⋯⋯,annoyed,,變數
liss,click_too_much,speech,主人慢一點～(*ˊᗜˋ*) 莉絲擔心您 最愛的主人♡,worried,,
liss,drag,speech,啊～主人～(´。• ᵕ •。`),happy,4000,drag 短台詞
```

**欄位說明**：
- `persona_id`: `haiyin` / `liss`
- `category`: `click_too_much` / `long_idle` / `continuous_use` / `deep_night` / `drag`
- `type`: `speech` / `thought` / `whisper`（drag/click 慎用 narration）
- `text`: 對白本體（含變數佔位 `{xxx}`）
- `expression`: 表情標籤（要在 persona.json `expressions` key 裡）
- `auto_close_ms`: 留空 = 用預設 12000；drag 通常 4000；thought 通常 5000-6000
- `note`: 備註（改寫來源、標記、靈感等，整理用）

### sequenceId 命名規則

`csv-to-dialogues.js` 會自動生成 sequenceId，格式：

```
{persona_id}_{cat_short}_{nnn}
```

`cat_short` 對照（保持跟 M3 範本一致）：

| category | cat_short |
|---|---|
| click_too_much | `ctm` |
| long_idle | `idle` |
| continuous_use | `cont` |
| deep_night | `night` |
| drag | `drag` |

範例：
- `haiyin_ctm_017`
- `haiyin_idle_042`
- `liss_night_103`
- `liss_drag_005`

`nnn` 為 3 位數連號（從 001 開始）。M3 範本佔 001-003，**M4 從 004 開始**。

## §5 風格自檢清單

每收完一輪 LLM 輸出，照這個清單對照：

### 海音（千春海音）

| 檢查項 | 通過標準 |
|---|---|
| **「中度病嬌」感** | 至少 30% 句子有佔有/嫉妒/獨佔感（「只看我一個」「跟誰」「比我重要嗎」） |
| **不極端** | 不出現殺戮 / 綁架 / 威脅生命的詞彙；「嗶~」屏蔽訓記 30 句最多 1-2 句 |
| **語助詞** | 「呐」「呢」「啦」「～」分散使用，不是每句都「呐」 |
| **稱謂** | 統一「我」/「你」，**不用「主人」** |
| **emoji** | minimal（30 句中 ❤ 控制在 6-10 句） |
| **內心戲比例** | thought 約佔 15-25%（depth_night / long_idle 較多） |
| **句型多樣** | 不是每句都是質問句 / 撒嬌句，要有獨白、抱怨、自戀混合 |

### 莉絲（莉茉黛拉絲）

| 檢查項 | 通過標準 |
|---|---|
| **「最愛的主人♡」結尾比例** | 60-80%（drag 例外，可降至 30-40%） |
| **顏文字頻率** | 30 句中至少 6-10 句有顏文字 |
| **常用顏文字使用** | (*ˊᗜˋ*) / (´。• ᵕ •。`) / (๑•̀ㅂ•́)و✧ 至少各出現 2-3 次 |
| **稱謂** | 統一「莉絲」/「主人」/「您」，**不用「你」** |
| **服侍意識** | continuous_use / deep_night 至少 25% 提到具體服侍動作（按摩 / 鋪被 / 泡茶） |
| **不冷淡** | 沒有命令式語氣（「請停止 X」這種） |
| **不嘮叨** | 連續 3 句同樣的關懷句型不能出現 |

### 整體（兩人格通用）

| 檢查項 | 通過標準 |
|---|---|
| **句長** | 每句中文 5-25 字（drag 5-12 字 / 一般 10-25 字） |
| **語氣多樣性** | 不要 30 句中 20 句都同樣句型 |
| **變數嵌入率** | 整體 < 30%；`{window_title}` 一份草稿 ≤ 2 句 |
| **type 比例** | speech 70% / thought 20% / whisper+narration 10% |
| **expression 多樣** | 至少用到 5 種以上不同 expression |
| **無 AI 自指** | 沒有「身為 AI / 作為語言模型 / 我是程式」這類詞彙 |
